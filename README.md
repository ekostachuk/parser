# CyberLeninka Scraper

Локальный Python-проект для поиска, отбора и выгрузки научных статей по нескольким источникам с browser-интерфейсом на Gradio и desktop-интерфейсом на Tkinter.

Проект уже умеет работать не только с `CyberLeninka`, но и с `eLIBRARY`, `OpenAlex`, `Crossref`, `Semantic Scholar`. Для `Google Scholar` в интерфейсе есть явный переключатель-напоминание, но автоматический сбор не включён, потому что у сервиса нет стабильного официального API для такого режима.

## Что умеет сейчас

- искать по нескольким ключевым словам и словосочетаниям через запятую;
- фильтровать по автору;
- принимать ручной список ссылок для парсинга;
- использовать несколько источников одновременно;
- выбирать несколько языков и несколько стран поиска в browser-версии;
- фильтровать по диапазону лет;
- учитывать слова-исключения;
- останавливать сбор с сохранением уже найденных результатов;
- сохранять:
  - основной результат;
  - отчёт по исключённым статьям;
  - список найденных, но неразобранных ссылок;
- выгружать данные в:
  - `XLSX` с 3 листами;
  - `CSV` с полным текстом;
  - `JSON` с полным текстом;
- считать `relevance_score`, тематические теги и пояснение `why_relevant`;
- показывать дашборды по:
  - источникам;
  - словам поиска;
  - популярным авторам.

## Источники поиска

- `CyberLeninka`
- `eLIBRARY`
- `OpenAlex`
- `Crossref`
- `Semantic Scholar`

## Browser UI

Browser-версия сейчас основная и самая удобная.

### Что есть в интерфейсе

- тема поиска;
- автор;
- слова-исключения;
- ручные ссылки;
- множественный выбор источников;
- множественный выбор языков поиска;
- множественный выбор стран поиска;
- диапазон лет;
- лимит статей для проверки;
- живая вкладка лога;
- вкладка результатов;
- вкладка дашбордов;
- кнопка принудительной остановки с выгрузкой текущего результата.

### Запуск browser-версии

```bash
pip install -r requirements.txt
python web_main.py
```

По умолчанию приложение откроется локально по адресу:

```text
http://127.0.0.1:8000
```

Если нужно поднять отдельную версию на другом порту:

```bash
GRADIO_SERVER_PORT=8006 python web_main.py
```

## Desktop UI

Desktop-версия тоже остаётся в проекте и подходит для работы из PyCharm.

```bash
pip install -r requirements.txt

```

## Форматы выгрузки

### XLSX

Excel-файл создаётся с тремя листами:

- `Кратко`
- `С абстрактом`
- `С полным текстом`

### CSV и JSON

- основной `CSV` содержит полный текст;
- основной `JSON` содержит полный текст;
- дополнительно создаются укороченные версии и версии с абстрактом;
- служебные отчёты сохраняются отдельно.

## Что попадает в выгрузку

В выгрузке уже есть:

- источник;
- название статьи;
- авторы статьи;
- год;
- журнал;
- область наук;
- аннотация;
- полный текст;
- `keywords` как конкретные слова или фразы поиска, по которым статья была найдена;
- `relevance_score`;
- `thematic_tags`;
- `why_relevant`;
- ссылка в тексте;
- ссылка для списка литературы по ГОСТ;
- DOI;
- ISSN;
- признак ВАК.

## Логика ранжирования

Автоматический `relevance_score` считается так:

- `title` = x3
- `keywords` = x2
- `abstract` = x2
- `full_text` = x1

Итоговая сортировка поднимает выше:

- статьи из источников с признаком `ВАК`;
- статьи с более высоким `relevance_score`;
- более свежие публикации, особенно `2020+`.

pip install -r requirements.txt

## Windows `.exe`

В проекте подготовлена сборка Windows `.exe` через GitHub Actions и локальный PyInstaller-сценарий.

Локальная сборка на Windows:

```bash
pip install -r requirements-build.txt
python scripts/build_exe.py
```

Результат:

```text
dist/CyberLeninkaParser.exe
```

## Важные замечания

- `eLIBRARY` обрабатывается осторожнее и медленнее, чем остальные источники;
- проблемные страницы складываются в отдельный отчёт `найдено, но не разобрано`;
- старые черновики `cyberleninka_app/exporter.py` и `cyberleninka_app/runner.py` не используются в рабочем приложении.
